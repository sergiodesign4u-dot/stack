Картка товару

Сторінка, заради якої існує довіра в магазині, і ціль конверсії кожної картки товару. Тут користувач перевіряє, що продукт безпечний, сертифікований і правильно дозований (Job 4: склад · дозування · походження · сертифікація), і робить крок «У кошик» (конверсія Job 2 і Job 3). На цій же сторінці живуть 3.1 Відгуки, 3.2 Питання, 3.3 Схожі. Принцип №1 — спершу прибрати сумнів, потім продавати.

01б · Діалоги — залишити відгук і поставити запитання

Два діалоги картки товару (3.1a · 3.2a)

03 · SEO-блок сторінки (A–E)

SEO: мета-теги, заголовки, текст, схема, чек-лист

Шаблон A–E для картки товару. Ключове для PDP: схема Product + Offers (ціна/наявність) і канонікалізація варіантів, щоб не плодити тонкі дублі.

A · Мета-теги

Доставка Новою Поштою, оплата картою."> (≤155) <link rel="canonical" href="https://stack.ua/protein/gold-standard-whey/"> (варіанти смак/вага → canonical на базовий товар) <link rel="alternate" hreflang="uk … ru … x-default"> · robots index,follow <meta property="og:type" content="product"> · og:title · og:image · product:price:amount · product:availability

E · Чек-лист оптимізації

  • Єдиний H1; кравлені хлібні крихти; лінк бренду → 2.4.
  • Ціна й наявність у схемі Product/Offers синхронні з реальним станом (вкл. out-of-stock).
  • Сертифікат — реальний контент (переглядне зображення/PDF), не лише текст — довіра + SEO.
  • Галерея: alt (товар + бренд + смак); головне фото не lazy (LCP), мініатюри — lazy.
  • Варіанти canonical на базу — без дублів від параметрів смаку/ваги.
  • Відгуки рендеряться серверно (кравлений текст), не лише через JS.
  • Core Web Vitals; українська мова контенту; таргети ≥ 44px.

05 · Зафіксовано / відкрите

✓ Зафіксовано (чернетка, 2026-06-30):

  • Таб-навігація по товару після брекрамсів (Про товар · Склад · Дозування · Характеристики · Відгуки · Питання); при скролі прилипає під стики-хедером (без мета-рядка) і стає його частиною, справа — ціна (зі знижкою/без) + «Купити» + ♡.
  • Ціна: стара (закреслена) зверху → нова крупно, бейдж −% поруч із ціною (не під нею — економія простору).
  • Блоки Доставка + Оплата під ціною/CTA в buy-box: НП (відділення/поштомат), кур'єр на адресу, самовивіз (Одеса, адреса [?]); ціни від 50 / 70 / безкоштовно; самовивіз — лише за наявності на точці. Оплата: картка, накладений платіж, готівка при самовивозі.
  • Єдина смуга довіри (оригінал · сертифікат · доставка · повернення) одним блоком перед складом (винесено з buy-box).
  • Блок довіри ведучий (Склад + фото етикетки складу · Дозування · Походження · Сертифікація) одразу за смугою; секції відкриті за замовч., можна згорнути; опис — після (Принцип №1).
  • Опис — повний продуктовий текст + додаткові фото продукту.
  • Один головний CTA «У кошик» + липка нижня смуга на мобілці; ♡ — вторинний. Проста відповідь під H1 (Принцип №5).
  • Варіанти на одній PDP, один canonical; відсутні варіанти вимкнено, не сховано.
  • Чесна наявність без таймерів; out-of-stock → «Повідомити» + повернення до колекції (не глухий кут).
  • Відгуки (3.1): ім'я + рейтинг + дата зліва, текст справа; категорії Переваги / Недоліки (можуть бути порожні); фото; відповіді (показати/сховати, з датами, можна фото); відповідь магазину — спец-чипс із логотипом замість імені. Два стани: багато → пагінація, мало → без неї. Сертифікат + відгуки = recovery конверсії (потік Job 3).
  • Діалог «Залишити відгук» (3.1a): модалка — рейтинг (обов'язковий) · переваги/недоліки (опц.) · текст · фото до 5 · ім'я; гість → авторизація (1.x) → повернення в діалог; сабміт → модерація → у стрічку 3.1.
  • Питання (3.2): «Поставити запитання» = діалог-форма (3.2a): лише текст + ім'я + опц. «сповістити про відповідь»; питання → у стрічку Q&A; відповідає магазин (чипс) або покупці; без рейтингу і без переваг/недоліків.
  • Статті по темі (блог 8.0/8.1) — секція перед футером: допомога новачку + SEO.
  • Характеристики дзеркалять фасети catalog.md; схожі (3.3) на тій самій сторінці.
  • Coach-aware: CTA «Додати клієнту» + ціна тренера. SEO A–E: Product/Offers + AggregateRating + Review + Breadcrumb; канонікалізація варіантів.

Лишається [?] (операційне / дані, не IA)

  • Контент на товар — склад, дозування, опис, файли сертифікатів: етап даних (реальні дані постачальника — операційний [?] із брифу).
  • Відгуки / рейтинг — потрібні реальні відгуки; агрегований рейтинг залежить від обсягу.

  • Правило ціни за порцію коли фасування/порції різняться — підтвердити на етапі даних.

  • Наявність на точці самовивозу — облік залишків по точках [?].
