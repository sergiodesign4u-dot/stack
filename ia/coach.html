Кабінет тренера

PRIMARY-джоба (Job 1, Рішення 1). Тренер — це покупець, що замовляє для клієнтів, а не об'єкт для перегляду. Глибокий за природою робочий флоу: «Для тренерів» лендинг (5.0) → активація ролі + верифікація + тариф (5.1) → кабінет (5.2) → клієнти (5.3/5.4) → мультиклієнтська сесія замовлення (5.5) з per-client групуванням у кошику. Тренер додає товари через швидке додавання в сесії, а не через глобальний пошук. Лендинг — публічний (SEO); робоча зона — приватна (noindex).

Вхідні двері тренера + тарифи Free / Pro

Публічна SEO-сторінка (front door тренера з хедера й футера). Ціннісна пропозиція під персону Олену: швидко зібрати замовлення для багатьох клієнтів за одну сесію, оптова ціна, надійна поставка. 10 · Зафіксовано / відкрите

Зафіксовано

  • Coach-as-buyer (Рішення 1): збережені клієнти + тег на клієнта + історія по клієнту. Не маркетплейс/каталог тренерів.
  • Роль активується поверх акаунта (не окремий логін); вхід — з лендингу 5.0 або акаунта 7.7.
  • Кабінет = шелл акаунта в режимі тренера + велика дія «Нова сесія».
  • Quick-add у сесії, не глобальний пошук; ціна тарифу тренера на кожному рядку.
  • Кошик групується по клієнтах (6.0). Глибина флоу — за природою (робоча), не сплющуємо.
  • Recovery, не глухі кути: OOS → заміна/пропустити; ціна тарифу не застосована → збережено, оформлення блоковано; рядок без клієнта → призначити/відкинути.
  • Лендинг індексується, робоча зона noindex.

Відкриті [?]

  • Тарифи Free/Pro: ціна Pro, ліміт клієнтів Free (2–3?), розмір опт-знижки — юніт-економіка й «ціновий гейт» тренера.
