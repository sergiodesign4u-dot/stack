Кабінет тренера

PRIMARY-джоба (Job 1, Рішення 1). Тренер — це покупець, що замовляє для клієнтів, а не об'єкт для перегляду. Глибокий за природою робочий флоу: «Для тренерів» лендинг (5.0) → активація ролі + верифікація + тариф (5.1) → кабінет (5.2) → клієнти (5.3/5.4, редагування 5.4a) → мультиклієнтська сесія замовлення (5.5) з per-client групуванням у кошику, далі — історія й деталі замовлень тренера (5.6/5.7). Тренер додає товари через швидке додавання в сесії, а не через глобальний пошук. Лендинг — публічний (SEO); робоча зона — приватна (noindex).

05 · Мультиклієнтська сесія замовлення (5.5) — ★ ядро Job 1

Замовлення для кількох клієнтів за одну сесію

Головний диференціатор. Вкладки клієнтів угорі + панель активного клієнта (його цілі → швидке додавання в сесії → рядки з ціною тарифу тренера + кількість + тег на клієнта) + підсумок сесії (сабтотал по кожному клієнту + разом). Товари додаються quick-add у сесії, не глобальним пошуком. Далі → кошик із групуванням по клієнтах (6.0).

Петля по клієнту (flows.md, Job 1): обрати/додати клієнта → quick-add (вантажить склад + ціну тарифу) → склад? немає → заміна / пропустити рядок → ціна тарифу? не застосувалась → сесію збережено, оформлення заблоковано → ще клієнти? → Кошик (per-client) → Оформлення. Активна вкладка = клієнт, тож рядок автоприв'язується (немає стану «без клієнта»). 08 · Деталі замовлення тренера (5.7)

Одне замовлення, згруповане по клієнтах

Одне розміщене замовлення, згруповане по клієнтах (те саме групування, що й у кошику тренера 6.0): шапка (№ · дата · статус · сума) → одна доставка на тренера + факти оплати → секції по клієнтах (клієнт · ціль · рядки з ціною тарифу тренера vs закреслений роздріб · сабтотал) → загальний підсумок. Дії: «↻ Повторити замовлення» → кошик 6.0, «Повторити для клієнта X» (per-client, Job 4), «Відстежити», назад до 5.6.

13 · Зафіксовано / відкрите

Зафіксовано

  • Coach-as-buyer (Рішення 1): збережені клієнти + тег на клієнта + історія по клієнту. Не маркетплейс/каталог тренерів.
  • Роль активується поверх акаунта (не окремий логін); вхід — з лендингу 5.0 або акаунта 7.7.
  • Кабінет = шелл акаунта в режимі тренера + велика дія «Нова сесія».
  • Quick-add у сесії, не глобальний пошук; ціна тарифу тренера на кожному рядку.
  • Кошик групується по клієнтах (6.0). Глибина флоу — за природою (робоча), не сплющуємо.
  • Recovery, не глухі кути: OOS → заміна/пропустити; ціна тарифу не застосована → збережено, оформлення блоковано; новий клієнт без позицій → quick-add; порожня сесія → додати першого клієнта.
  • Лендинг індексується, робоча зона noindex.

Відкриті [?]

  • Тарифи Free/Pro: ціна Pro, ліміт клієнтів Free (2–3?), розмір опт-знижки — юніт-економіка й «ціновий гейт» тренера.
  • Оптова ціна на launch-обсязі — чи взагалі прохідна для тренера (стратегія v5, головний ризик).
  • Верифікація соц-лінку — ручна/автоматична, хто модерує — операційний процес.
  • Тарифи доставки / вибір платіжного провайдера для кошика тренера (6.x, per-client групування вже в page-level — cart.md).
