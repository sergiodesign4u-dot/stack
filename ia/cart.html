Кошик і оформлення

Точка сходження обох флоу. Сюди веде і картка товару (покупець), і мультиклієнтська сесія тренера (5.5). Кошик — швидкий діалог-drawer (для тренера — групування по клієнтах); оформлення — контакт → доставка → оплата → перегляд; далі стан «Замовлення оформлено» → історія (7.2). Тут застосовуються бонуси й лояльність покупця та тариф тренера. Транзакційна зона — noindex.

02 · Оформлення (6.1)

Контакт → доставка → оплата → перегляд

Одна сторінка з секціями (не багатокроковий майстер): контакт (телефон — вхід за кодом, якщо гість) → доставка (НП відділення/поштомат · кур'єр · самовивіз Одеса; місто+відділення через діалог 0.1a; збережені адреси 7.5) → оплата → підсумок справа (липкий). Тренер бачить розбивку по клієнтах у підсумку.

05 · Стани

Стани кошика й оформлення

СтанПоведінка
Порожній кошикEmpty-state 🛒 + «До каталогу» / «За ціллю».
Гість на оформленніТелефон + код (passwordless) — акаунт створюється авто; далі одразу оформлення.
Немає збережених адресЗахоплюємо адресу в секції доставки; пропонуємо зберегти (→ 7.5).
Товар став OOS у кошикуПозначаємо рядок, пропонуємо прибрати / замінити; не даємо оформити недоступне.
Оплата відхиленаНазад у кошик/оплату (retry), не термінал; замовлення не втрачається.
Самовивіз, товару немає на точціМетод недоступний для цих позицій → підказка обрати доставку.
Тренер: групуванняКошик і підсумок — по клієнтах; одна доставка тренеру; розбивка зберігається в замовленні.
Бонуси / лояльністьПокупець: знижка авто + бонуси перемикачем. Тренер: тариф на рядках, без персональних бонусів.
Завантаження / підтвердженняСкелетони; після оплати — стан 6.2 + SMS-підтвердження.
06 · SEO / приватність

Транзакційна зона — noindex, без схеми

Кошик та оформлення — транзакційні, не контентні сторінки (як акаунт і авторизація). Виняток із A–E.

АспектРішення
robotsnoindex, nofollow на /cart, /checkout, /order/{id}
SchemaНемає (транзакційні дані). Order-confirmation — приватна
URL/cart (діалог також на будь-якій сторінці) · /checkout · /order/{id}
АналітикаКлючова воронка: add-to-cart → checkout → purchase (події для Mixpanel/PostHog)
A11y / довіраКроки як fieldset/legend; помилки полів озвучені; бейджі оплати/повернення; жодних таймерів (принцип #4)

08 · Зафіксовано / відкрите

Зафіксовано

  • Кошик = діалог-drawer (швидкий перегляд, не забирає зі сторінки); порожній стан веде в каталог/цілі.
  • Тренер — групування по клієнтах (із сесії 5.5); одна доставка тренеру, розбивка зберігається в замовленні.
  • Оформлення однією сторінкою (секції контакт/доставка/оплата + липкий підсумок), не багатокроковий майстер.
  • Доставка: НП відділення/поштомат · кур'єр · самовивіз Одеса; місто через діалог 0.1a; адреси з 7.5.
  • Оплата: LiqPay/Wayforpay (карта) · Apple/Google Pay · накладений · готівка при самовивозі.
  • Passwordless-checkout: гість підтверджує телефон кодом → акаунт створюється авто.
  • Бонуси/лояльність покупця тут; тариф тренера — на рядках. Recovery: оплата назад у кошик, OOS не оформлюється.
  • Транзакційна зона noindex; спокійно, без таймерів (принцип #4).

Відкриті [?]

  • Знижка лояльності / бонуси: % , курс, чи сумуються зі знижками — з 7.4 [?].
  • Тариф доставки: точні суми НП/кур'єр — за розцінками перевізника (інтеграція).
  • Платіжний провайдер: LiqPay vs Wayforpay — рішення технічного скоупінгу.
  • Мін. сума / поріг безкоштовної доставки — маркетинг/операційно [?].
  • Тренер: чи можлива розбивка доставки по клієнтах у майбутньому (зараз — одна) [?].
