SEO-методика та ключові слова

Рушій SEO проєкту: принципи, модель ключів, кластери запитів під спортпит, модифікатори інтенту, шаблони Title/Description/H1, структура SEO-тексту, схема й технічний чек-ліст. З цим можна генерувати готову, оптимізовану українську копію для будь-якої сторінки вже на етапі IA — чекає лише валідація обсягів за реальними даними. Живить A–E SEO-блок кожної сторінки, матрицю категорій та SEO-лінки футера.

01 · Принципи

Принципи SEO для Stack

  • Довіра передусім (Принцип №1): спершу безпека/якість (сертифікати, оригінал, склад, дозування), потім продаж — знижує сумнів початківця.
  • Проста українська (Принцип №5): щоденні слова в навігації; глибина — у тілі тексту. Кореневі ключі природно, без переспаму.
  • Збіг інтенту: сторінка цілить в один домінантний інтент (комерційний/інформаційний) та один основний ключ; решта — LSI, не повтор.
  • Один H1 на сторінку, логічна ієрархія H2→H3.
  • Внутрішня перелінковка описовими анкорами (категорії ↔ субкатегорії ↔ цілі ↔ бренди ↔ міста). Футер і мега-меню — дві головні поверхні лінкування.
  • Керування фасетною індексацією: індексуємо лише білий список фасет-сторінок, решта — canonical-на-базу / noindex,follow (проти розпухання індексу).
  • Двомовність (uk/ru): користувачі шукають і українською, і російською. Сайт uk-first (canonical uk) + `ru` hreflang; без змішування мов на одному URL.
  • Технічна гігієна: mobile-first, Core Web Vitals, alt, lazy-load, чисті URL, sitemap.xml, крихти.
02 · Модель ключів та модифікатори інтенту

Head · Body · Long-tail + слова-модифікатори

Модель ключів
  • Head (широкі, високочастотні): протеїн · креатин · вітаміни → H1 категорії.
  • Body (комерційні): купити протеїн · протеїн ціна · сироватковий протеїн → Title/Description + субкатегорії.
  • Long-tail (вузькі, високий інтент/інфо): протеїн без лактози купити · як приймати креатин → фільтри/цілі/товар/блог.
Модифікатори інтенту (UA) — комбінуємо з head-словом
Комерційні купити · ціна · вартість · в Україні · [місто] · оригінал · дешево · знижка · акція · замовити · доставка
Вибір/якість найкращий · рейтинг · відгуки · який обрати · топ
Інформаційні як приймати · дозування · для чого · користь · протипоказання · коли пити · скільки

05 · Шаблони Title / Description / H1

Шаблони мета-тегів

Title (≤60): [Head] — [купити/ціна] [в Україні] | Stack → Креатин — купити креатин моногідрат, ціна | Stack Description (≤155): [Що є / типи] від перевірених брендів. [Оригінал/сертифікати]. [Доставка/оплата]. [Ціль/CTA]. → Креатин моногідрат, Creapure, HCl у порошку й капсулах — для сили й маси. Оригінал, сертифікати, доставка Новою Поштою. H1 (єдиний): гола назва сутності (Креатин) — без «купити» в H1 Ціль (2.2) H1: Спортивне харчування для [цілі] Місто (2.1a) H1: [Категорія] [Місто] + локальний рядок доставки
06 · Структура SEO-тексту

Тіло тексту (блок унизу сторінки)

  1. Вступ (довіра) — що це за категорія + сигнал якості (1–2 речення).
  2. Як обрати — типи/сценарії простими словами (цінність для початківця), кореневі ключі природно.
  3. Завершення (довіра/логістика) — сертифікація, доставка Новою Поштою, оплата, повернення.

Обсяг ~600–1200 символів; щільність ключів ~1–2%; LSI/синоніми, не повтор. Розташування — під товарами+фільтрами; «Читати більше» зверху веде якорем сюди.

08 · Технічний чек-ліст

Технічне SEO

  • canonical на кожній; пагінація self-canonical; фасет-комбінації canonical-на-базу.
  • hreflang uk / ru / x-default; без змішування мов на URL.
  • Чисті слаги: /protein/ · /protein/isolate/ · /brand/optimum-nutrition/.
  • XML-sitemap (категорії, субкатегорії, товари, бренди, міста, блог) + robots.txt.
  • Крихти (BreadcrumbList) на всіх глибоких сторінках.
  • Зображення: описовий alt, lazy-load нижче згину, стиснені; LCP не блокується.
  • Core Web Vitals (LCP/CLS/INP) у нормі; mobile-first.
  • Внутрішні лінки — реальні кравлені <a> (load-more не ховає пагінацію).
09 · Інструменти та процес

Як генеруємо SEO-текст сторінки

Інструменти (обсяги [?])
AhrefsSerpstat (UA)Google Keyword PlannerSearch ConsoleGoogle Trends (uk vs ru)
  1. Визначаємо тип сторінки → її патерн основного ключа (§03).
  2. Тягнемо кластер категорії (§04) + модифікатори інтенту (§02).
  3. Заповнюємо шаблони Title/Description/H1 (§05).
  4. Пишемо тіло за 3-частинною структурою (§06), довіра-перш, без переспаму.
  5. Додаємо схему (§07) + проганяємо технічний чек-ліст (§08).
  6. Пізніше валідуємо обсяги/складність інструментами; правимо формулювання, не структуру.

Лишається [?]

  • Реальні обсяги / складність / конкуренти в видачі — дані інструментів (Ahrefs/Serpstat/GKP).
  • Розподіл uk-vs-ru запитів за кластером — Search Console / Trends після запуску.
  • Фінальний білий список фасетної індексації — keyword research.
  • План тем блогу (інформаційний long-tail) — контент-етап.
