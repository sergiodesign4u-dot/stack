Системні сторінки та глобальні компоненти

Крос-функційні елементи, потрібні на всьому сайті: сторінки помилок і глобальні компоненти. Тон — спокійний (Принцип 4, без таймерів) і завжди один зрозумілий наступний крок (жодних глухих кутів).

Сторінки станів

HTTP 404 · noindex
Сторінку не знайдено
Повноцінна сторінка з хедером/футером: пошук + швидкі посилання + рекомендовані товари. Ніколи не soft-404 (200). Не глухий кут.

HTTP 500 · noindex
Помилка сервера
Мінімальний шаблон, незалежний від бекенду (він сам може лежати): лого + «Оновити» / «На головну» / «Підтримка».
Відкрити 500 →
HTTP 503 · Retry-After
Технічні роботи
Запланований стан, не збій. 503 + Retry-After — щоб пошук трактував як тимчасове, а не зникнення сайту. Без зворотного таймера.

Cookie-банер (Закон України «Про захист персональних даних»)

Cookie ставимо лише після згоди (бездіяльність ≠ згода). Три рівнозначні дії — «Прийняти всі» / «Тільки необхідні» / «Налаштувати»; відмова така ж проста, як згода. Категорії Аналітика/Маркетинг вимкнені за замовчуванням (opt-in), Необхідні — завжди активні. Банер живий унизу сторінки; «Змінити згоду» також є у футері.

Тости / сповіщення (глобальний компонент)

Легкий зворотний зв'язок для дій: додано в кошик, збережено адресу, підписка, відгук. Три типи — ok / error / info, авто-зникнення (~4 c) + закриття, aria-live="polite". Критичні рішення — діалог, не тост. Без спаму.

Видалений товар — 301 чи 410 [?]

Коли товар прибрали назавжди: 301 на категорію, до якої він належав (краще за 404), або 410 Gone, якщо його не буде взагалі. Це операційне правило — фіналізується з реальними даними каталогу. Локалізація системних сторінок (uk/ru) — за загальним i18n.
